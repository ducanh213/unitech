# BÁO CÁO TỔNG HỢP KHỐI LƯỢNG CÔNG VIỆC VÀ KIẾN TRÚC KỸ THUẬT DỰ ÁN UNITECH
(Phiên bản nâng cao dành cho Đồ án Tốt nghiệp xuất sắc)

Dự án UniTech không chỉ dừng lại ở mức độ ứng dụng quản lý cơ bản (CRUD) mà đã được kiến trúc, thiết kế và lập trình theo tiêu chuẩn của một phần mềm doanh nghiệp (Enterprise Software) với độ phức tạp cao, chú trọng vào bảo mật, hiệu năng và tính toàn vẹn dữ liệu.

---

## PHẦN 1: TỔNG HỢP CÔNG VIỆC FRONTEND (GIAO DIỆN & TRẢI NGHIỆM NGƯỜI DÙNG)
Frontend được xây dựng bằng React.js, tập trung vào việc xử lý khối lượng lớn state (trạng thái) và phân luồng người dùng phức tạp.

1. Kiến trúc Component & Tối ưu Hiệu năng (Performance Optimization):
- Triển khai Lazy Loading (React.lazy & Suspense) và Code Splitting: Chia nhỏ các bundle JS để giảm thời gian tải trang ban đầu (FCP), đặc biệt quan trọng với hệ thống có nhiều phân hệ (Admin, Teacher, Student) như UniTech.
- Xây dựng Custom Hooks: Tách biệt hoàn toàn logic (fetching data, form handling) khỏi UI components để tái sử dụng mã nguồn.
- Áp dụng Higher-Order Components (HOC) và Protected Routes: Xây dựng lớp màng bọc bảo mật phía client, chặn đứng việc truy cập trái phép bằng cách kiểm tra Token và Role (Vai trò) ngay trên trình duyệt trước khi render giao diện.

2. Quản lý Trạng thái & Tương tác API (State Management & API Integration):
- Cấu hình Axios Interceptors: Đánh chặn (intercept) mọi request gửi đi để tự động đính kèm JWT Bearer Token, và đánh chặn response trả về để bắt lỗi toàn cục (Global Error Handling), ví dụ: tự động đá user ra trang login nếu Token hết hạn (Error 401 Unauthorized).
- Quản lý đồng bộ UI: Sử dụng các kỹ thuật cập nhật UI lạc quan (Optimistic UI updates) khi sinh viên đăng ký môn học để tăng độ phản hồi, sau đó đồng bộ lại với trạng thái thực từ server.

3. Xây dựng 3 Phân hệ Độc lập với luồng nghiệp vụ sâu:
- Admin Portal: Xử lý các Data Grid (Bảng dữ liệu) lớn với hàng ngàn bản ghi, tích hợp phân trang (Pagination) và tìm kiếm/lọc đa chiều (Multi-dimension filtering). Xây dựng Dashboard biểu đồ thống kê trực quan.
- Teacher Portal: Thiết kế giao diện nhập điểm dạng Spreadsheet (bảng tính) thời gian thực, tự động tính toán tổng điểm trên UI để giảng viên đối chiếu trước khi submit về server.
- Student Portal: Xây dựng thuật toán lọc môn học theo Đợt đăng ký (Period), cảnh báo trùng lịch, và giao diện xem lịch học thông minh (Timetable View).

## PHẦN 2: TỔNG HỢP CÔNG VIỆC BACKEND & CSDL (LINH HỒN CỦA HỆ THỐNG - TRỌNG TÂM ĐIỂM CAO)
Backend (Node.js/Express.js & MongoDB) được thiết kế theo mô hình Modular Monolith, xử lý các bài toán hóc búa về tính đồng thời (Concurrency) và an toàn dữ liệu.

1. Thiết kế Hệ thống Bảo mật Đa lớp (Multi-layer Security Architecture):
- Authentication & Authorization: Mã hóa mật khẩu một chiều bằng thuật toán Hash Bcrypt. Quản lý phiên bằng JSON Web Token (JWT) có gán thời gian sống (Expiration). Xây dựng chuỗi Middleware chặn quyền truy cập API theo Role một cách nghiêm ngặt.
- Data Sanitization & Validation: Mọi luồng dữ liệu (Payload) từ Frontend gửi lên đều được làm sạch và kiểm tra tính hợp lệ chặt chẽ, ngăn chặn triệt để các cuộc tấn công NoSQL Injection và Cross-Site Scripting (XSS).
- Chống Brute Force & Rate Limiting: (Kỹ thuật nâng cao) Bảo vệ các endpoint nhạy cảm (như Login, Quên mật khẩu) khỏi các cuộc tấn công dò mật khẩu liên tục.

2. Xử lý Logic Nghiệp vụ & Thuật toán Phức tạp (Complex Business Logic):
- Bài toán Race Condition (Tình trạng tương tranh) trong Đăng ký học phần: 
  * Đây là tính năng cốt lõi và khó nhất. Khi hàng ngàn sinh viên cùng đăng ký 1 lớp học có sĩ số giới hạn (Ví dụ: 40 chỗ) tại cùng một thời điểm, hệ thống phải xử lý đồng bộ để không bị vượt quá sĩ số. 
  * Thuật toán Backend phải tuần tự hóa: Kiểm tra khóa (Lock) -> Đọc sĩ số hiện tại -> Nếu < 40 thì Ghi nhận Đăng ký -> Tăng sĩ số lên 1. Mọi bước được gói gọn để đảm bảo tính ACID (Toàn vẹn dữ liệu) như một Transaction.
- Hệ thống Phân loại và Tính điểm tự động (Grading Engine):
  * Tách biệt logic tính điểm ra khỏi Frontend. Server là nơi duy nhất giữ công thức: 10% Chuyên cần + 30% Giữa kỳ + 60% Cuối kỳ.
  * Tự động làm tròn số (Rounding algorithms) theo tiêu chuẩn học thuật, và tự động ánh xạ sang điểm chữ (A, B, C, D, F) khi kết thúc môn.
- Kiểm soát trạng thái vòng đời (Lifecycle Status): Quản lý vòng đời của Đợt đăng ký (Period) (Mở - Đang chạy - Đóng). Nếu Period bị đóng, mọi API ghi nhận đăng ký mới lập tức bị vô hiệu hóa ở tầng Backend, bất kể Frontend có gửi request hay không.

3. Tối ưu hóa Database (MongoDB Performance Tuning):
- Schema Design (Thiết kế Schema): Áp dụng triệt để cơ chế Reference (Tham chiếu ObjectId) thay vì Embedding (Nhúng) cho những dữ liệu có tính biến động cao (như Lớp học, Điểm số) để tránh phình to CSDL (Document Size Limit).
- Đánh Chỉ mục (Database Indexing): Các trường dữ liệu thường xuyên bị query như `studentId`, `classCode`, `email` được đánh Index (Chỉ mục) giúp tốc độ tìm kiếm (Lookup) giảm từ mức độ $O(n)$ xuống $O(log n)$, tăng tốc độ phản hồi API lên gấp nhiều lần khi dữ liệu lớn.
- Multi-level Aggregation & Populate: Viết các câu lệnh Query gom nhóm phức tạp (Aggregation Pipeline) để kết xuất dữ liệu thống kê cho Admin Dashboard chỉ bằng 1 lần chọc xuống CSDL, giảm độ trễ (Latency) của mạng.

4. Kiến trúc Mã nguồn và Xử lý Bất đồng bộ (Clean Architecture & Async Processing):
- Tách biệt rõ ràng Route (Bộ định tuyến) - Controller (Bộ điều khiển logic) - Model (Bộ giao tiếp dữ liệu).
- Xử lý Bất đồng bộ (Asynchronous/Promises): Toàn bộ thao tác I/O (Lưu Database, Gửi Email OTP) đều xử lý bất đồng bộ non-blocking, giúp Server Node.js (dù chạy đơn luồng) vẫn chịu tải (Concurrent requests) cực kỳ ấn tượng.
- Global Error Handler: Bắt và định dạng chuẩn (Format) toàn bộ các lỗi ném ra (Exception) trong hệ thống về một định dạng JSON duy nhất { status, error_code, message }, giúp Frontend dễ dàng xử lý.

## PHẦN 3: TÍCH HỢP TRÍ TUỆ NHÂN TẠO (AI/ML) VÀO HỆ THỐNG BACKEND (ĐIỂM NHẤN ĐỘT PHÁ)
Để nâng tầm dự án từ một hệ thống quản lý thông thường thành một hệ thống thông minh, dự án đã tích hợp thành công một Module Trí tuệ nhân tạo chạy độc lập (Microservice Architecture).

1. Kiến trúc Microservice cho AI Module:
- Thay vì nhồi nhét code AI vào Node.js (vốn không mạnh về tính toán ma trận), hệ thống đã tách riêng một service bằng Python sử dụng framework FastAPI. Việc này thể hiện tư duy thiết kế hệ thống phân tán (Distributed System) cực kỳ chuyên nghiệp. Backend Node.js và AI Python giao tiếp với nhau qua chuẩn REST API.
- Sử dụng các thư viện tính toán khoa học hàng đầu: `pandas` để tiền xử lý (Preprocessing) tập dữ liệu, và `scikit-learn` để xây dựng mô hình học máy.

2. Thuật toán Học máy (Machine Learning Algorithm):
- Triển khai thuật toán Random Forest: Hệ thống không dùng các thuật toán cơ bản mà sử dụng Random Forest (Học máy tập hợp - Ensemble Learning) để phân tích dữ liệu lịch sử điểm số của sinh viên từ database MongoDB. 
- Random Forest giúp dự đoán kết quả học tập, từ đó xây dựng logic "Hệ thống Gợi ý Học phần" (Course Recommendation System) giúp sinh viên chọn môn học phù hợp với năng lực, hoặc tạo ra các biểu đồ cảnh báo rủi ro trượt môn (nếu có) cho quản trị viên.

3. Luồng Xử lý Dữ liệu (Data Pipeline):
- Dữ liệu thô (Raw data) về sinh viên, khóa học và điểm số được trích xuất từ MongoDB, làm sạch (loại bỏ null/duplicate), chuyển hóa thành ma trận đặc trưng (Feature Engineering) và đưa vào Model đã được train (lưu trữ bằng Joblib) để đưa ra dự đoán (Prediction).
